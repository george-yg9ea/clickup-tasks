# ClickUp Tasks Dashboard

A personal dashboard that displays ClickUp tasks assigned to you, with project grouping, subtask accordions, status filters, and sortable columns.

**Tech stack:** Next.js 16, React 19, TypeScript, shadcn/ui, Tailwind CSS 4, TanStack Table, NextAuth.js v5 (Google OAuth), ClickUp API v2.

---

## Quick start (local development)

```bash
# 1. Clone the repo
git clone https://github.com/george-yg9ea/clickup-tasks.git
cd clickup-tasks

# 2. Install dependencies
npm install

# 3. Set up environment variables (see below)
cp .env.example .env.local
# Edit .env.local with your values

# 4. Generate an auth secret
npx auth secret
# This writes AUTH_SECRET to .env.local automatically

# 5. Run the dev server
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

## Environment variables

Create a `.env.local` file in the project root with these values:

```env
# Auth
AUTH_SECRET=           # Auto-generated by `npx auth secret`
AUTH_GOOGLE_ID=        # Google OAuth Client ID
AUTH_GOOGLE_SECRET=    # Google OAuth Client Secret

# ClickUp
CLICKUP_API_TOKEN=     # Your ClickUp personal API token
CLICKUP_TEAM_ID=       # Your ClickUp workspace (team) ID
```

### How to get each value

#### `AUTH_SECRET`
Run `npx auth secret` in the project root. It generates a random secret and writes it to `.env.local`.

#### `AUTH_GOOGLE_ID` and `AUTH_GOOGLE_SECRET`
See [docs/GOOGLE_OAUTH_SETUP.md](docs/GOOGLE_OAUTH_SETUP.md) for a step-by-step guide. In short:
1. Go to [Google Cloud Console → Credentials](https://console.cloud.google.com/apis/credentials)
2. Create a project (or use an existing one)
3. Set up the OAuth consent screen
4. Create an **OAuth 2.0 Client ID** (Web application)
5. Add authorized redirect URIs:
   - Local: `http://localhost:3000/api/auth/callback/google`
   - Production: `https://your-domain.vercel.app/api/auth/callback/google`
6. Copy the **Client ID** and **Client Secret**

#### `CLICKUP_API_TOKEN`
1. Open ClickUp → click your avatar (bottom-left) → **Settings**
2. Go to **Apps** (in the sidebar)
3. Under **API Token**, click **Generate** (or copy your existing token)

#### `CLICKUP_TEAM_ID`
This is your ClickUp Workspace ID. To find it:
1. Open ClickUp in your browser
2. Look at the URL: `https://app.clickup.com/12345678/...`
3. The number after `app.clickup.com/` is your Team (Workspace) ID

## Customization

To make this your own, update these files:

### App title (browser tab + navbar)

**Browser tab title** — `app/layout.tsx`:
```typescript
export const metadata: Metadata = {
  title: "Your Name's Tasks",  // ← change this
};
```

**Navbar heading** — `components/navbar.tsx`:
```typescript
<h1 className="text-xl font-semibold">Your Name's Tasks</h1>  // ← change this
```

### Allowed email domain

By default, only `@whiterabbit.group` emails can sign in. Change this in `auth.ts`:
```typescript
const ALLOWED_EMAIL_DOMAIN = "whiterabbit.group";  // ← change to your domain
```

To allow any email, remove the domain check in the `signIn` callback entirely.

## Deploy to Vercel

1. Push your fork/clone to GitHub
2. Go to [vercel.com/new](https://vercel.com/new) and import your repo
3. Add all environment variables from above in **Settings → Environment Variables**
4. Deploy

After deployment, update your Google OAuth credentials to include the production URL:
- **Authorized JavaScript origins:** `https://your-domain.vercel.app`
- **Authorized redirect URIs:** `https://your-domain.vercel.app/api/auth/callback/google`

Every push to `main` auto-deploys to production.

## Features

- **Group by Project** — tasks grouped under expandable project accordions, sorted by most recent activity
- **All Tasks** — flat sortable list with project column
- **Subtask tree** — nested accordions for subtasks at any depth (task → subtask → sub-subtask → …)
- **Status filters** — show/hide Done, Backlog, and Recurring tasks (hidden by default)
- **Sortable columns** — click any column header to sort
- **Clickable links** — task names, project names, and list names link to ClickUp
- **Google sign-in** — restricted to a configurable email domain
